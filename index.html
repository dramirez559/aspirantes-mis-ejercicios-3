S3 - Git Branches

Ejercicio 1:

  1. Crea un repositorio de Git vacío con el comando git init.
  2. Crea un archivo llamado index.html y agrega algo de contenido.
  3. Agrega el archivo index.html al área de preparación con el comando git add index.html.
  4. Crear un commit en este punto.
  5. Crea una nueva rama llamada develop con el comando git branch develop.
  6. Cambia a la rama develop con el comando git checkout develop.
  7. Crea un archivo llamado .env y agrega algo de contenido sensible.
  8. Crea un archivo llamado .gitignore en la raíz del repositorio.
  9. Abre el archivo .gitignore y agrega una línea que indique que Git debe ignorar el archivo .env. La línea debería ser así: .env.
  10. Agrega el archivo .gitignore al área de preparación con el comando git add .gitignore.
  11. Edita el archivo index.html para hacer algunos cambios y agrega esos cambios al área de preparación con el comando git add index.html.
  12. Haz un commit con los cambios en la rama develop usando el comando git commit -m "Cambios en la rama develop".
  13. Cambia de vuelta a la rama main con el comando git checkout main.
  14. Muestra el contenido del archivo index.html para comprobar que los cambios hechos en la rama develop no están presentes.
  15. Fusiona la rama develop en la rama main usando el comando git merge develop. Los cambios realizados en la rama develop se integrarán en la rama main.
  16. Muestra el contenido del archivo index.html de nuevo para comprobar que los cambios realizados en la rama develop ahora están presentes en la rama main.
  17. Verifica que el archivo .env no está incluido en el repositorio ejecutando el comando git ls-files --other --ignored --exclude-standard.
  18. Publicar a Github en un repositorio llamado aspirantes-mir-ejercicioi-3.
  19. Enviar el link al repositorio en Github a su mentor(a).

Bonus

Este ejercicio te permite practicar cómo clonar un repositorio existente, cómo crear una nueva rama a partir de la rama principal y cómo fusionar los cambios realizados en la nueva rama de vuelta a la rama principal.

  1. Clona un repositorio existente con el comando git clone .
  2. Cambia al directorio del repositorio clonado con el comando cd .
  3. Crea una nueva rama llamada nueva-funcionalidad a partir de la rama principal con el comando git checkout -b nueva-funcionalidad. Este comando crea una nueva rama y automáticamente cambia a ella.
  4. Crea un nuevo archivo llamado nuevo-archivo.txt en el directorio raíz del repositorio y agrega algo de contenido.
  5. Agrega el archivo nuevo-archivo.txt al área de preparación con el comando git add nuevo-archivo.txt.
  6. Haz un commit con los cambios en la rama nueva-funcionalidad usando el comando git commit -m "Agregado nuevo archivo para la nueva funcionalidad".
  7. Cambia de vuelta a la rama principal con el comando git checkout main
  8. Muestra el contenido del archivo nuevo-archivo.txt para comprobar que los cambios realizados en la rama nueva-funcionalidad no están presentes.
  9. Fusiona la rama nueva-funcionalidad en la rama main usando el comando git merge nueva-funcionalidad. Los cambios realizados en la rama nueva-funcionalidad se integrarán en la rama main.
  10. Muestra el contenido del archivo nuevo-archivo.txt de nuevo para comprobar que los cambios realizados en la rama nueva-funcionalidad ahora están presentes en la rama main.
